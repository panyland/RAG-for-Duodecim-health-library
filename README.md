# MDAI – Terveyskirjasto RAG Assistant

A Retrieval-Augmented Generation (RAG) application for querying the Finnish health library [Terveyskirjasto](https://www.terveyskirjasto.fi), published by Duodecim. Ask medical questions in Finnish or English and get answers grounded in the article database.

## How it works

```
Terveyskirjasto website
        |
   [1] main.py          — scrapes all articles and saves them as markdown files
        |
   [2] vector_db.py     — chunks the markdown, embeds it with a multilingual model,
        |                  and stores it in a local ChromaDB vector database
        |
   [3] app.py           — Streamlit chat UI: takes your question, retrieves the 5
                           most relevant article chunks, and sends them as context
                           to an LLM (Groq / llama-3.3-70b) for a grounded answer
```

The embedding model (`paraphrase-multilingual-MiniLM-L12-v2`) supports Finnish natively, so both the indexing and retrieval steps handle Finnish text correctly.

The app uses Groq to run `llama-3.3-70b-versatile`. Groq's free tier allows 14,400 requests/day.

1. Sign up at [console.groq.com](https://console.groq.com)
2. Go to **API Keys** → **Create API Key**
3. Copy the key (starts with `gsk_...`)

## Setup

### 1. Clone the repo and install dependencies

```bash
git clone <repo-url>
cd RAG-for-Duodecim-health-library
pip install uv        # if you don't have uv
uv sync
```

### 2. Configure your API key

Copy the example env file and fill in your Groq key:

```bash
cp .env.example .env
```

Open `.env` and replace `your_groq_api_key_here` with your actual key. This file is listed in `.gitignore` and will never be committed.

### 3. Scrape the articles (skip if you already have `terveys_markdown/`)

This downloads all articles from Terveyskirjasto as markdown files. It takes a while depending on your connection.

```bash
uv run python main.py
```

### 4. Index the vector database

This must be run before the app, and re-run any time the embedding model changes. It reads `terveys_markdown/`, splits the articles by heading structure, embeds them, and writes the result to `terveys_chroma_db/`.

```bash
uv run python vector_db.py
```

Indexing takes roughly 15–30 minutes on CPU for the full article set (~11,000 files).

### 5. Launch the app

```bash
uv run streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## Project structure

```
.
├── main.py             # Article scraper (crawl4ai)
├── vector_db.py        # Embedding + ChromaDB indexing
├── chat.py             # RAG chain (retriever + LLM)
├── app.py              # Streamlit UI
├── terveys_markdown/   # Scraped articles (git-ignored, generated by main.py)
├── terveys_chroma_db/  # Vector database (git-ignored, generated by vector_db.py)
├── .env                # Your secrets (git-ignored, copy from .env.example)
└── .env.example        # Template showing required environment variables
```
